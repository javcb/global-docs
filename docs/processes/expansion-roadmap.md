# Expansion roadmap

## Principle

The global-docs pattern (centralized markdown, AI-readable, context-portable)
can apply to any domain that benefits from documented rules and processes.

## Potential future domains

- personal-ops: family logistics, tutoring plans, household rules
- finance-ops: QuickBooks tagging rules, budget categories, expense policies
- business-ops: client onboarding, project templates, SOPs

## Rules for expansion

- Only create a new domain repo when you have real, immediate content to add
- Use the same folder structure: README with index, subfolders by topic
- Use the same AI-INSTRUCTIONS.md pattern pointing back to domain-specific rules
- Do not pre-build empty domains; placeholders here are enough until needed
